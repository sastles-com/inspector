# scratch-inspector 🔍

Raspberry Pi と PiCamera2、および KIZKI アルゴリズムを用いた自動傷検査システムです。

## 🚀 概要
本プロジェクトは、人間の視覚生理機構（周辺視・固視微動）をモデル化した **KIZKI アルゴリズム** を実装し、画像内の微細な傷や欠陥を「ポップアウト（浮き彫り）」させるシステムです。

### 主な機能
- **多重解像度 KIZKI**: 複数のスケール（ブロックサイズ）を統合し、細い傷から大きなムラまで同時に検出。
- **白黒両対応**: 背景平均からの絶対値乖離を計算することで、明るい傷（白線）と暗い傷（黒線）の両方を検出可能。
- **比較暗合成 (Dark Synthesis)**: 複数の照明条件の画像から、影や傷の候補を効率的に統合。
- **Streamlit GUI**: パラメータをリアルタイムで調整し、ヒートマップで結果を確認可能。

## 🛠 セットアップ
本プロジェクトは `uv` を使用してパッケージ管理を行っています。

```bash
# 依存関係のインストール
uv sync
```

## 💻 使い方

### 1. Streamlit アプリの起動（GUI）
ブラウザ上でパラメータを調整しながら傷検出を試せます。

```bash
uv run streamlit run streamlit_kizki.py
```

### 2. 比較暗合成ツールの実行（CUI）
ディレクトリ内の全画像を比較暗（各画素の最小値）で合成します。

```bash
uv run src/dark_synthesis.py data/your_session_dir/ -o result.png
```

### 3. テスト用合成画像の生成
点や線の傷をシミュレートした画像を生成して、アルゴリズムを検証できます。

```bash
uv run src/generate_test_images.py
```

## 📂 プロレクトリ構成
- `src/`: コアロジック (KIZKI 検出器、合成ツールなど)
- `data/`: 撮影データおよび合成サンプル画像
- `tests/`: 単体テストコード
- `streamlit_kizki.py`: メインの GUI アプリケーション
